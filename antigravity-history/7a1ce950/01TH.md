# Refactor Plan: Admin Home & Tournament Sorteo Flow

## Goal Description
The objective is to refine the Admin Home screen and the Tournament generation flow to better align with the actual operations workflow. 
Specifically:
1. Update `AdminHomeScreen` layout (adjust titles, remove redundant buttons).
2. Update `TournamentWizard` to only handle creation of the tournament structure without automatically assigning players or generating matches.
3. Update `AdminScreen` to remove redundant inline configuration inputs (as the wizard already captures them), and introduce a clear two-step flow: 
    - **Step 1:** View the empty structure. 
    - **Step 2:** Execute a "Sortear" (Draw) action to automatically assign players to groups/brackets and then generate the corresponding fixtures (matches).

## Proposed Changes

### 1. Database Fixes (SQL snippet for User)
Since the production database might not have the latest local migrations, we will output a raw SQL snippet that the Admin can run in their Supabase console to fix the creation bug:
- Re-create `categories_level_check` allowing `'Escalafon'`, `'Primera'`, etc.
- Alter `rr_group_members` to drop `NOT NULL` on `user_id`. This allows us to insert empty placeholder rows for groups.

### 2. `src/screens/AdminScreen.tsx` (Master-Detail Flow)
- **Master View (Torneos Creados)**: Render *only* a list of `tournaments`. Provide the "Nuevo Campeonato" button here which opens the `TournamentWizard`.
- **Detail View**: When a tournament is tapped, set `selectedTournamentId`. This will render the details for that specific tournament:
  - The Categories within that tournament.
  - The "Sorteo" (Sortear Jugadores) action.
  - The Generated Matches and Groups ready for assignment.
- We will consolidate Data fetching (`loadData`) to fetch tournaments first. When a tournament is selected, fetch its context.

### 3. Empty Group Members (Visualizing placeholders)
- When completing the wizard for Round Robin, we will insert empty rows into `rr_groups` AND insert `playersPerGroup` number of empty rows into `rr_group_members` (with `user_id: null` and sequential `sort_order`).
- Update `PlayerAssignModal` to support assigning a player to an empty `rr_group_members` row.

## Verification Plan
### Manual Verification
- The user will apply the raw SQL to fix the constraints.
- Test that Admin View loads just a list of Tournaments initially.
- Test that selecting a tournament shows its details.
- Test that Tournament Wizard creates empty group placeholders that can be tapped to assign players.
