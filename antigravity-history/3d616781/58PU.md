# 📱 ArriendoCanchas — App Móvil con Capacitor

## Contexto

ArriendoCanchas es una plataforma web construida con **Next.js 16 + Supabase + TailwindCSS**, desplegada en Dokploy con Docker (`output: 'standalone'`). El objetivo es crear apps nativas para **Android e iOS** que compartan la misma base de datos y lógica de negocio.

---

## User Review Required

> [!IMPORTANT]
> **Decisión arquitectónica clave: Live URL vs Static Export**
>
> Tu proyecto usa **19 API routes**, **middleware SSR** con autenticación/roles, y **server components**. Capacitor normalmente requiere páginas estáticas (HTML/CSS/JS sin servidor). Hay dos caminos:
>
> | Enfoque | Qué implica | Tiempo | Riesgo |
> |---------|-------------|--------|--------|
> | **Live URL** (recomendado ahora) | La app carga tu URL de producción dentro de un WebView nativo + plugins nativos (push, splash, etc.) | 1-2 días | Bajo |
> | **Static Export** | Reescribir las 19 API routes como llamadas directas a Supabase desde el cliente, eliminar SSR | 3-4 semanas | Alto (puede romper la web) |
>
> **Mi recomendación**: Empezar con **Live URL** para tener la app funcionando rápido, y luego migrar progresivamente a static export en una fase posterior si se necesita offline o más velocidad.

### Fase 2: Firebase + Push Notifications

#### Supabase — Nueva tabla `push_tokens`

```sql
CREATE TABLE push_tokens (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  token TEXT NOT NULL,
  platform TEXT NOT NULL CHECK (platform IN ('android', 'ios', 'web')),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, token)
);

ALTER TABLE push_tokens ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can manage their own tokens"
  ON push_tokens FOR ALL
  USING (auth.uid() = user_id);
```

#### [NEW] `src/lib/capacitor/pushNotifications.ts`
- Lógica de registro de tokens de dispositivo
- Listener para notificaciones recibidas
- Función para guardar token en tabla `push_tokens` de Supabase

#### Supabase Edge Function — `send-push-notification`
- Recibe `user_id` + `title` + `body`
- Busca tokens del usuario en `push_tokens`
- Envía vía Firebase Cloud Messaging (FCM)

#### Supabase Edge Function — `booking-reminders`
- Cron job que revisa reservas próximas (ej: 2 horas antes)
- Envía recordatorio push al usuario
- Revisa pagos pendientes y envía notificación

## Verificación

### Testing en Desarrollo
1. **Android Studio Emulator**: Instalar Android Studio, crear un emulador, correr `npx cap open android` y probar la app
2. **Dispositivo físico Android**: Conectar por USB, habilitar depuración USB, instalar el APK directamente
3. **iOS Simulator** (requiere Mac): Correr `npx cap open ios` en Xcode

### Testing Manual (necesito tu ayuda)
1. **Abrir la app** → Debe cargar la landing page de ArriendoCanchas
2. **Login** → Debe funcionar igual que en la web
3. **Crear una reserva** → Flujo completo, verificar que aparece en la web también
4. **Push Notifications** → Verificar que llega el recordatorio de reserva al dispositivo
5. **Botón atrás (Android)** → Debe navegar correctamente (no cerrar la app)
6. **Orientación** → La app debe funcionar en portrait y landscape
7. **Cruce de datos** → Crear reserva en web, verificar que aparece en app y viceversa
