# Test Plan: Standardized Booking Status Flow

This document details the manual test steps to verify the removal of 'RECHAZADA', the correct mapping of statuses, and the 10-minute expiration rule.

## 1. Verify Status Removal ('RECHAZADA')

### Admin Panel
- [ ] Go to `/admin/bookings`.
- [ ] Open any booking to edit.
- [ ] Open the "Estado" dropdown.
- [ ] **Expectation**: 'RECHAZADA' should **NOT** be visible in the list.
- [ ] **Expectation**: Options should include: Pendiente Pago, En Verificación, Pagada, Cancelada, Bloqueada.

### Frontend Code
- [ ] Check `src/components/ui/badge.tsx` (Completed via code review).
- [ ] Check `src/components/admin/BookingEditModal.tsx` (Completed via code review).

## 2. Verify Status Mapping (Admin -> User)

### Scenario A: Admin creates "Pendiente Pago"
1. **Admin**: Create a booking with status `PENDIENTE_PAGO`.
2. **User**: Log in as a regular user (or the user assigned to the booking).
3. **User**: Go to `/bookings` (Mis Reservas).
4. **Expectation**: 
   - Status badge should say **"Hold"** (active) or **"Expirada"** (if >10 mins).
   - If < 10 mins: Badge color Yellow/Amber.
   - If > 10 mins: Badge color Red (Expirada).

### Scenario B: Admin creates "Bloqueada"
1. **Admin**: Create a booking with status `BLOQUEADA` (Purple).
2. **User**: Go to `/dashboard` or `/bookings`.
3. **Expectation**: 
   - If assigned to user: Status should map to **"Cancelada"** or not appear in "Next Booking" if we treat it as non-active.
   - **Crucial**: Check `/availability` (Public View).
   - **Expectation**: The slot should appear as **Purple/Blocked** (not just generic gray).

### Scenario C: Admin creates "Pagada"
1. **Admin**: Create a booking with status `PAGADA`.
2. **User**: View in `/bookings`.
3. **Expectation**: Status badge says **"Pagada"** (Green).

### Scenario D: Admin creates "Cancelada"
1. **Admin**: Create a booking with status `CANCELADA`.
2. **User**: View in `/bookings`.
3. **Expectation**: Status badge says **"Cancelada"** (Gray/Red).

## 3. Verify HOLD Expiration (10 Minutes)

### Test Steps
1. **User/Admin**: Create a booking with status `PENDIENTE_PAGO`.
2. **Immediate Check**: 
   - Go to public availability (`/availability`).
   - Slot should be **unavailable/booked**.
3. **Wait 11 Minutes** (or manipulate DB time).
4. **Delayed Check**:
   - Go to public availability (`/availability`).
   - **Expectation**: Slot should be **AVAILABLE** again (logic in `availability/week` filters out expired pendings).
   - **User Dashboard**: In `/bookings`, the specific booking should show as **"Expirada"**.
